# Scaffold Guide — Task 4용 시맨틱 마크업 가이드

> 용도: Task 4 (HTML Scaffolder)가 `content.json`을 시맨틱 HTML로 변환할 때 따라야 할 **DOM 구조와 클래스명 규칙**.
>
> ⚠️ 이 단계에서는 **CSS 한 줄도 작성하지 않습니다.** 디자인은 Task 5의 영역.
> ⚠️ 콘텐츠 텍스트는 **단 한 글자도 변형하지 않습니다.** content.json 그대로.
> ⚠️ 클래스명은 `upstage-design-tokens.md` §4의 컴포넌트 매핑이 후킹할 수 있는 이름으로.

## 1. 페이지 전체 골격

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{{canonical_name}} 가이드북 — Trend Tech Guidebook</title>
  <meta name="description" content="{{one_liner}}">

  <!-- Google Fonts (Task 5에서 채워짐) -->
  <!-- 인라인 CSS (Task 5에서 채워짐) -->
  <style>/* INLINE_CSS_PLACEHOLDER */</style>
</head>
<body>

  <header class="site-header">
    <div class="header-inner">
      <a href="#top" class="logo">{{canonical_name}}</a>
      <nav class="header-nav">
        <a href="#toc">목차</a>
        <a href="{{official_url}}" target="_blank" rel="noopener">공식</a>
      </nav>
    </div>
  </header>

  <main id="top">
    <article class="guidebook">

      <!-- 가이드북 헤더 -->
      <header class="guide-header">
        <p class="section-label">Trend Tech Guidebook</p>
        <h1>{{canonical_name}}</h1>
        <p class="lede">{{one_liner}}</p>
        <p class="meta">
          <span class="badge badge--purple">생성: {{generated_at}}</span>
          <span class="badge badge--gray">언어: 한국어</span>
        </p>
      </header>

      <!-- 디스앰비기 명시 (필요한 경우) -->
      <aside class="callout callout--note">
        <span class="callout-icon">ℹ️</span>
        <div class="callout-body">
          <p>이 가이드는 <strong>{{canonical_name}}</strong> ({{official_url}})을 다룹니다. 같은 이름의 다른 도구가 있다면 별도 가이드를 참조하세요.</p>
        </div>
      </aside>

      <!-- 목차 -->
      <nav class="toc" id="toc">
        <p class="toc-title">목차</p>
        <ol>
          <!-- 각 블록의 섹션을 순서대로 -->
          <li><a href="#section-00">00. 시작하기 전에</a></li>
          <li><a href="#section-01">01. {{canonical_name}}란?</a></li>
          <!-- ... -->
        </ol>
      </nav>

      <!-- 본문 섹션 (블록별로 묶음) -->
      <section class="block" data-block-id="1" id="block-1">
        <header class="block-header">
          <p class="section-label">Block 1</p>
          <h2>진입 결정</h2>
        </header>

        <section class="guide-section" id="section-00">
          <header class="section-header">
            <p class="section-label">Section 00</p>
            <h3>시작하기 전에</h3>
            <p class="intro-promise">{{intro_promise}}</p>
          </header>

          <div class="section-body">
            <!-- content.json의 body 배열을 변환 (아래 §3 참조) -->
          </div>

          <footer class="section-footer">
            <p class="outro-summary"><strong>한 줄 요약:</strong> {{outro_summary}}</p>
          </footer>
        </section>

        <!-- 블록 1 안의 다른 섹션들 -->
      </section>

      <!-- 다음 블록들 -->

      <!-- 가이드북 푸터 -->
      <footer class="guide-footer">
        <h2>다음 단계 / 더 읽을거리</h2>
        <ul class="source-list">
          <!-- 모든 출처 모음 -->
        </ul>
        <p class="generated-by">Generated by trend-tech-guidebook-builder · {{generated_at}}</p>
      </footer>

    </article>
  </main>

</body>
</html>
```

## 2. 주요 컨테이너 클래스 명명

| 영역 | 클래스 / 태그 |
|---|---|
| 사이트 헤더 | `<header class="site-header">` |
| 가이드북 article | `<article class="guidebook">` |
| 블록 (1~7) | `<section class="block" data-block-id="N">` |
| 섹션 (00~17) | `<section class="guide-section" id="section-NN">` |
| 섹션 헤더 | `<header class="section-header">` |
| 섹션 본문 | `<div class="section-body">` |
| 섹션 푸터 | `<footer class="section-footer">` |
| 가이드 푸터 | `<footer class="guide-footer">` |
| 목차 | `<nav class="toc">` |
| 사이드 라벨 | `<p class="section-label">` |
| 도입 약속 | `<p class="intro-promise">` |
| 마무리 요약 | `<p class="outro-summary">` |

## 3. content.json `body` 항목 → HTML 변환 규칙

content.json의 `body` 배열에 들어가는 각 객체 타입별 변환:

### 3-1. paragraph

```json
{ "type": "paragraph", "text": "...", "_sources": ["url1", "url2"] }
```

→

```html
<p>...
  <sup class="source-refs">
    <a href="url1" class="source-ref" target="_blank" rel="noopener">[1]</a>
    <a href="url2" class="source-ref" target="_blank" rel="noopener">[2]</a>
  </sup>
</p>
```

출처 번호는 가이드 전체에서 **고유한 ID 부여 + 푸터 출처 모음 리스트와 매핑**.

### 3-2. callout

```json
{ "type": "callout", "variant": "tip|note|warn|danger", "text": "...", "icon": "💡" }
```

→

```html
<aside class="callout callout--{variant}">
  <span class="callout-icon" aria-hidden="true">{icon}</span>
  <div class="callout-body">
    <p>{text}</p>
  </div>
</aside>
```

### 3-3. table

```json
{ "type": "table", "headers": ["col1", "col2"], "rows": [["a", "b"], ["c", "d"]], "highlight_col": 1 }
```

→

```html
<div class="table-wrap">
  <table>
    <thead>
      <tr><th>col1</th><th>col2</th></tr>
    </thead>
    <tbody>
      <tr><td>a</td><td class="highlight-col">b</td></tr>
      <tr><td>c</td><td class="highlight-col">d</td></tr>
    </tbody>
  </table>
</div>
```

`highlight_col`이 명시된 경우 해당 열에 `class="highlight-col"`.

### 3-4. code

```json
{ "type": "code", "lang": "bash", "code": "claude code init" }
```

→

```html
<pre><code class="language-bash">claude code init</code></pre>
```

신택스 하이라이트 클래스는 highlight.js / Prism 호환 명명 (`language-{lang}`). Task 5에서 사전-하이라이트 처리.

### 3-5. heading_inline (섹션 안의 작은 헤딩)

```json
{ "type": "heading", "level": 4, "text": "Step 1. 설치" }
```

→

```html
<h4>Step 1. 설치</h4>
```

### 3-6. list

```json
{ "type": "list", "ordered": true, "items": ["...", "..."] }
```

→

```html
<ol>
  <li>...</li>
  <li>...</li>
</ol>
```

또는 unordered면 `<ul>`.

### 3-7. quote (인용)

```json
{ "type": "quote", "text": "...", "_sources": ["url"], "author": "@user" }
```

→

```html
<blockquote>
  <p>...</p>
  <cite>— @user, <a href="url" target="_blank" rel="noopener">출처</a></cite>
</blockquote>
```

### 3-8. comparison (나쁜 예 → 좋은 예)

```json
{
  "type": "comparison",
  "bad": { "label": "❌ 나쁜 예", "text": "..." },
  "good": { "label": "✅ 좋은 예", "text": "..." }
}
```

→

```html
<div class="comparison">
  <div class="comparison-item comparison-item--bad">
    <p class="comparison-label">❌ 나쁜 예</p>
    <blockquote><p>...</p></blockquote>
  </div>
  <div class="comparison-item comparison-item--good">
    <p class="comparison-label">✅ 좋은 예</p>
    <blockquote><p>...</p></blockquote>
  </div>
</div>
```

### 3-9. qa (Q&A 항목)

```json
{
  "type": "qa",
  "question": "...",
  "cause": "...",
  "solution": ["방법1", "방법2"],
  "code": "..."
}
```

→

```html
<div class="qa">
  <h4 class="qa-question">Q. ...</h4>
  <p class="qa-cause"><strong>원인:</strong> ...</p>
  <p><strong>해결:</strong></p>
  <ol class="qa-solution">
    <li>방법1</li>
    <li>방법2</li>
  </ol>
  <pre><code>...</code></pre>
</div>
```

### 3-10. step (튜토리얼 단계)

```json
{
  "type": "step",
  "n": 1,
  "title": "설치",
  "command": "brew install x",
  "explanation": "왜 이렇게 하는지 설명",
  "tip": "💡 팁: ..."
}
```

→

```html
<div class="step">
  <h4 class="step-title"><span class="step-num">Step 1.</span> 설치</h4>
  <pre><code class="language-bash">brew install x</code></pre>
  <p class="step-explanation">왜 이렇게 하는지 설명</p>
  <aside class="callout callout--tip">
    <span class="callout-icon">💡</span>
    <div class="callout-body"><p>팁: ...</p></div>
  </aside>
</div>
```

### 3-11. before_after (Before/After 사례)

```json
{
  "type": "before_after",
  "situation": "...",
  "before": { "label": "기존 방식 (30분)", "steps": ["...", "..."] },
  "after":  { "label": "X 방식 (3분)",   "command": "..." }
}
```

→

```html
<div class="before-after">
  <p class="ba-situation"><strong>상황:</strong> ...</p>
  <div class="ba-grid">
    <div class="ba-col ba-col--before">
      <p class="ba-label">기존 방식 (30분)</p>
      <ol>...</ol>
    </div>
    <div class="ba-col ba-col--after">
      <p class="ba-label">X 방식 (3분)</p>
      <pre><code>...</code></pre>
    </div>
  </div>
</div>
```

## 4. 출처 매핑 (Footnote 스타일)

가이드북 전체에서 출처 URL은 **번호로 인덱싱**해서 본문 인라인 + 푸터 모음 양쪽에 표시.

```html
<!-- 본문 -->
<p>...주장... <sup class="source-refs"><a href="#src-1">[1]</a></sup></p>

<!-- 푸터 -->
<footer class="guide-footer">
  <h2>출처</h2>
  <ol class="source-list">
    <li id="src-1"><a href="https://..." target="_blank" rel="noopener">https://... — 페이지 제목</a></li>
    <li id="src-2">...</li>
  </ol>
</footer>
```

같은 URL이 본문 여러 곳에서 인용되면 **하나의 번호**만 부여.

## 5. 접근성 (Accessibility) 최소 규칙

- 모든 `<img>`에 `alt` (장식이면 `alt=""`)
- `<a>` 외부 링크는 `target="_blank" rel="noopener"`
- 콜아웃 아이콘 이모지는 `aria-hidden="true"`
- 헤딩 위계 `h1 → h2 → h3 → h4` 건너뛰지 않기
- `<main>` 한 번만, `<nav>`에 `aria-label` 부여

---

## 6. ID 명명 규칙 (앵커용)

- 블록: `id="block-{N}"` (1~7)
- 섹션: `id="section-{NN}"` (00~17)
- 출처: `id="src-{N}"`
- 페이지 최상단: `id="top"`
- 목차: `id="toc"`
