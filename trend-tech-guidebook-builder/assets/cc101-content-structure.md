## 1. 섹션 청사진 (20섹션 → 일반화한 7블록)

cc101 원본 섹션을 트렌드 기술 일반에 적용 가능한 **7개 의미 블록**으로 추상화. 새 키워드 가이드 만들 때 이 7블록을 채우면 됨.

```
[블록 1] 진입 결정 (Filter & Confidence)
  ├─ 섹션 0: "이게 나를 위한 건가?" (대상 사용자 명시 + 진입장벽 해소)
  └─ 섹션 1: "이게 뭐야 한 줄 정의" (핵심 가치 한 문장 + 비유)

[블록 2] 가치 인지 (Why Now)
  ├─ 섹션 2: "이걸로 뭘 할 수 있어" (활용 사례 5개+)
  └─ 섹션 3: "왜 지금 트렌드인가" (커뮤니티 반응·시그널)

[블록 3] 투자 결정 (Cost & Commit)
  └─ 섹션 4: 가격/라이센스/계정 (Free/Pro 비교, 하드웨어 요구사항 등)

[블록 4] 첫 경험 (First Win)
  ├─ 섹션 5: 설치 (OS별 단계)
  ├─ 섹션 6: 첫 실행 Step-by-step (5분 안에 "오 됐다" 경험)
  └─ 섹션 7: 좋은 프롬프트/사용법 vs 나쁜 예

[블록 5] 효율성 확보 (Daily Workflow)
  ├─ 섹션 8: 핵심 개념 (내부 동작 모델)
  ├─ 섹션 9: 설정 파일·메모리 (장기 사용 패턴)
  └─ 섹션 10: 명령어/단축키 레퍼런스

[블록 6] 문제 해결 (Real World)
  ├─ 섹션 11: "자주 막히는 곳" Q&A (10~15개)
  ├─ 섹션 12: 디버깅 워크플로우 (3단계)
  └─ 섹션 13: 협업/팀/공유 패턴 (있을 경우)

[블록 7] 고급화 (Power User)
  ├─ 섹션 14: 외부 연결 (API, MCP, 통합)
  ├─ 섹션 15: 자동화 (훅/스크립트/스킬)
  ├─ 섹션 16: 다른 도구 결합
  └─ 섹션 17: 다음 단계 / 더 읽을거리 (출처 모음)
```

**섹션 채택 우선순위**:
- **필수 (모든 가이드)**: 0, 1, 2, 5, 6, 11, 17
- **강력 권장**: 3, 4, 7, 8
- **선택 (주제에 맞으면)**: 9, 10, 12~16

새 키워드 가이드 만들 때 모든 17섹션을 다 채울 필요는 없음. **필수 7개 + 주제에 맞는 강력권장**만 채워도 OK. 단, **블록 순서는 절대 바꾸지 않음** (1→7).

## 2. 분량 배분 (블록별 비율)

| 블록 | 비율 | 의미 |
|---|---|---|
| 1+2 진입+가치 | 20% | 이탈 방지의 핵심 |
| 3+4 투자+첫경험 | 30% | "오 됐다" 순간이 가이드의 절정 |
| 5 효율성 | 20% | 반복 사용자 만들기 |
| 6 문제해결 | 15% | 현실 안전망 |
| 7 고급화 | 15% | 깊이 있는 사용자 인도 |

**핵심 원칙**: 첫 경험(블록 4)에 가장 많은 분량을 투입. 여기서 막히면 다 끝남.

## 3. 콘텐츠 패턴 6종 (재사용 가능)

각 섹션 안에서 콘텐츠를 구성할 때 쓸 수 있는 패턴. 여러 개 섞어 써도 됨.

### 패턴 A: 비교표 (Concept vs Concept)

```markdown
| 항목 | Option A | Option B |
|---|---|---|
| 하는 일 | ... | ... |
| 비유 | ... | ... |
| 언제 쓰나 | ... | ... |

→ 선택 기준: ...
```

**적용**: "X vs 기존 방식" / "Free vs Pro" / "Mode A vs Mode B"

---

### 패턴 B: Step-by-Step 튜토리얼

```markdown
### Step 1. [단계명]

`명령어 또는 액션`

[왜 이렇게 하는지 한 줄 설명]

> 💡 팁: ...
```

**적용**: 설치, 첫 실행, 협업 셋업

---

### 패턴 C: 나쁜 예 → 좋은 예

```markdown
**❌ 나쁜 예 (모호함)**
> "코드 개선해줘"

**✅ 좋은 예 (구체적)**
> "auth.js의 login 함수에서 이메일 정규식 검증이 빠져있어. RFC 5322 기준으로 추가하고 테스트도 같이 짜줘"
```

**적용**: 프롬프트 작성, 명령 사용, 설정 작성

---

### 패턴 D: 설정 파일 + 옵션표

````markdown
```yaml
---
name: 예시 설정
mode: production
---
key: value
```

**옵션 설명**

| 옵션 | 의미 | 기본값 |
|---|---|---|
| name | ... | ... |
| mode | ... | ... |
````

**적용**: config 파일, 메모리 파일, manifest

---

### 패턴 E: Q&A — 증상 → 원인 → 해결

```markdown
### Q. [증상 한 줄]

**원인**: [근본 이유]

**해결**:
1. [방법 1]
2. [방법 2]

`코드 또는 커맨드`
```

**적용**: "자주 막히는 곳" 섹션, 트러블슈팅

---

### 패턴 F: Before/After 사례

```markdown
**상황**: [구체 업무 예시]

**기존 방식** (30분)
1. ...
2. ...
3. ...

**X 방식** (3분)
> "한 줄 명령어 또는 한 번의 액션"
```

**적용**: 활용 사례 섹션, 가치 증명

## 4. 톤 & 보이스

| 항목 | 규칙 | 비율 |
|---|---|---|
| 어조 | 존댓말 기본 + 가벼운 반말 양념 | 80:20 |
| 비유 | 기술 개념 → 생활 비유 적극 | 섹션당 1~2회 |
| 이모지 | 카테고리/중요도 표시 (💡팁, ⚠️주의, ✅체크, ❌금지, 🔴Critical) | 섹션당 3~5개 |
| 공감 | "처음엔 다들 그렇습니다" 류의 격려 | 진입부에 필수 |
| 솔직함 | 단점·한계도 명시 | 가격·트레이드오프 섹션 |
| 강조 | **굵음**, 화살표(→), 박스, 색 강조 | 핵심에만 |
| 리스트 | 단계는 번호, 옵션은 불릿 | 일관성 |
| 단락 길이 | 3~5줄 이내 | 호흡감 |

**금지**:
- 오만한 톤 ("당연히 알겠죠?")
- 과장 ("혁명적인", "세상을 바꿀")
- 영어 직역체 ("이것은 ~을 가능하게 합니다")
- 한 단락 6줄 초과

**필수 표현**:
- 진입부: "처음 보면 어렵게 느껴질 수 있습니다" / "코딩을 몰라도 됩니다"
- 막힌 곳 안내: "여기서 막혔다면 → 다음 섹션 [링크]"
- 단점 명시: "단, [한계]는 있습니다"
- 마무리: "한 줄 요약: [핵심]"

---

## 5. 도입부 약속 패턴

각 섹션 상단(또는 가이드 최상단)에 **이 섹션이 무엇을 약속하는지** 한 문장으로 명시.

**공식**: `[사용자 상황] — [도구/방법]으로 [구체적 결과]`

예시:
- "처음 X를 접한 분 — 5분 안에 첫 결과물 만들기"
- "매번 같은 설명을 반복하던 분 — 설정 파일 한 번으로 영구 해결"
- "단순 도구로만 쓰던 분 — 자동화 시스템으로 발전시키기"

---

## 6. 마무리 패턴

각 섹션 말미는 **다음 섹션으로의 연결** + **한 줄 요약**.

```markdown
> **한 줄 요약**: [이 섹션의 핵심 한 문장]
>
> 다음: [막혔으면 → 트러블슈팅 / 더 알고 싶으면 → 고급 섹션]
```

가이드 전체 마무리(블록 7-17 "다음 단계"):
- 출처 링크 모음 (공식 / 튜토리얼 / 커뮤니티)
- "이제 [구체적 다음 행동]을 시도해보세요"
- 도움 받을 곳 (디스코드, 깃헙 이슈 등)

---

## 7. 초보자 심리 흐름 매핑

가이드를 따라가는 사용자가 단계별로 갖는 의문을 미리 해소하는 구조.

```
"이게 나한테 필요한가?"             → 블록 1 (섹션 0)
"이게 정확히 뭔데?"                 → 블록 1 (섹션 1)
"근데 왜 지금 다들 얘기해?"         → 블록 2 (섹션 3)
"돈은 얼마나 들어?"                 → 블록 3
"어떻게 깔아?"                       → 블록 4 (섹션 5)
"첫 번째로 뭘 해봐?"                → 블록 4 (섹션 6)
"내가 잘 쓰고 있는 건가?"           → 블록 4 (섹션 7)
"왜 가끔 이상하게 동작해?"          → 블록 5 (섹션 8)
"매번 같은 말 하기 싫은데?"         → 블록 5 (섹션 9)
"이거 안 되는데 어쩌지?"            → 블록 6 (섹션 11)
"다른 도구랑 같이 쓸 수 있어?"      → 블록 7 (섹션 14)
"더 자동화 할 수 있어?"             → 블록 7 (섹션 15)
"다음에 뭘 더 배워?"                → 블록 7 (섹션 17)
```

**Task 3 합성 시 체크리스트**: 위 13개 의문 중 가이드가 답해주지 않는 게 있는지 확인. 없으면 보강.

## 9. 새 키워드 가이드 합성 시 체크리스트

Task 3가 콘텐츠 합성을 끝냈을 때 다음을 자체 검증:

- [ ] 블록 1~7 순서를 지켰는가?
- [ ] 필수 7섹션(0,1,2,5,6,11,17)이 모두 있는가?
- [ ] 첫 경험(블록 4) 분량이 30% 안팎인가?
- [ ] 섹션마다 도입부 약속 + 마무리 한 줄 요약이 있는가?
- [ ] 패턴 A~F를 최소 4가지 이상 활용했는가?
- [ ] 비유가 섹션마다 1~2개 들어갔는가?
- [ ] 단점·한계를 솔직히 명시한 곳이 있는가?
- [ ] 모든 주장에 출처 링크가 attach되어 있는가?
- [ ] 한 단락이 5줄을 초과하는 곳이 없는가?
- [ ] 13개 초보자 의문 중 답해주지 않은 게 있는가?
