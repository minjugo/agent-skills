# 진행 파일 구조 (빈 워크시트)

시작할 때 `progress/{팀명}_진행.md`를 아래 구조로 만들고, 세션마다 해당 섹션을 채워 갱신하세요.
이 파일이 *대화로 채워지는 워크시트의 원본*이며, 마지막에 HTML UI와 PRD .docx의 입력이 됩니다.

```markdown
# AI2W 워크시트 진행 — {팀명}
- 팀명:
- 소속/팀원:
- 작성일:
- 진행 상태: (예: WS3 진행 중)

## WS1 · 자동화하고 싶은 업무 (큰 그림)
- 자동화하고 싶은 업무(한 문장):
- 왜(지금 답답한 점):
- 누가·언제·빈도:
- 현재 대략의 모습:
- 기대 변화:

## WS2 · As-Is 업무 Flow
- 시작 트리거:
- 최종 산출물:
- 단계(한 줄 한 행위, 태그 [P]병목 [H]사람필수 [◎]Agent가능):
  1.
  2.
  3.
- Pain Point 메모:

## WS3 · To-Be(AX) · Agent 로드맵
- To-Be 흐름(단계 순서, Agent 박스는 [AGENT]로 표시, 사람 판단은 [HUMAN]):
  1.
  2.
- 로드맵의 Agent 목록(만들고 싶은 것 전부):
  - A.
  - B.
  - C.

## WS4 · Agent 우선순위
- 후보별 (효과 / 실현가능성 / 순위):
  - A:
  - B:
- ▶ 1순위 Agent:
- 고른 이유(1줄):

## WS5 · Agent 정의 (1순위)
- 이름:
- 목적(1문장):
- 수행 단계(①→②→③):
- 트리거:
- 기대 산출물:
- Human 개입(단독 / HITL 루프):
- Tool(단계별):
- Input(항목·출처·형식):
- Output(항목·저장위치·형식):
- 구조(단독/보조 sub/멀티):

## WS6 · PRD 초안
- 1. 개요(이름/목적/가치):
- 2. 업무 맥락(As-Is + Pain Point 3):
- 3. ★대상 사용자(누가·언제·어디서/빈도):
- 4. 선정 이유:
- 5. ★기능 요구사항(기능 단위 + 트리거·I/O):
- 6. Human 개입 설계:
- 7. ★구현 방향(도구·데이터·연동·준비 데이터):
- 8. 기대효과 & 측정지표(정량/정성/KPI):
- 9. 리스크 & 전제:
```

★ = 실사용을 좌우하는 핵심 칸(대상 사용자·기능 요구사항·구현 방향).
