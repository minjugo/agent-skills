# 순서도 도형 사용 규칙 (범용)

> 이 규칙은 **어떤 업무에도 동일하게** 적용된다(특정 task에 비특화). 퍼실리테이터가 As-Is(WS2)·To-Be(WS3)를 그릴 때, 그리고 HTML 렌더 시 이 규칙을 따른다.

## 도형 ↔ 의미

| 도형 | CSS class | 의미 | 언제 쓰나 |
|---|---|---|---|
| **타원** ◯ | `node start` | 시작 / 종료 | 흐름의 첫 트리거와 최종 완료 지점 |
| **사각형** ▭ | `node process` | 처리 단계 | 사람·시스템이 수행하는 **하나의** 행위 |
| **평행사변형** ▱ | `node io` | 입력 / 출력 | 흐름에 들어오거나 나가는 데이터·문서·자료 |
| **마름모** ◇ | `node decision` | 판단 / 분기 | **Yes/No로 갈라지는** 의사결정 지점 |
| **화살표** ▼ | `conn` | 진행 방향 | 단계 간 순서·방향 |
| **보라 박스**(To-Be) | `node agent` | Agent 처리 구간 | As-Is에서 ◎로 표시한 자리를 교체 |

## 작성 규칙

1. **한 박스 = 한 행위.** "~하고 ~한다"면 둘로 분리한다.
2. **숨은 분기는 마름모로.** "~인 경우 / ~면 / ~인지 확인 / 승인? / 확정?"처럼 갈림이 들어있으면 처리 박스가 아니라 **마름모**다.
3. **마름모는 반드시 Yes·No 두 갈래의 '다음 목적지'를 표기.** 한쪽만 그리지 않는다.
4. **되돌아가면 복귀(loop)로.** 한 갈래(보통 No)가 앞 단계로 돌아가면 `↩ ○○ 단계로 복귀`라고 적는다.
5. **산출물이 2개면 평행사변형도 2개**로 분리한다.
6. **동시 진행은 '병렬'로 묶는다**(`parallel`).
7. **태그 뱃지**(P 병목 / H 사람필수 / ◎ Agent가능)는 처리·입출력 박스에 붙인다. 마름모(판단)는 대개 사람 판단이라 H 성격이지만, 뱃지 없이 도형 자체로 의미를 전한다.

## 예시 — 분기 + 복귀

```
        ( 시작 )
           ▼
   [ 신청·요청 접수 ]
           ▼
        ◇ 요건 충족?
        ├ Yes → 처리 진행
        └ No  → 보완 요청 단계로 복귀 ↩
           ▼ (Yes)
   [ 처리 진행 ] H
           ▼
         ( 종료 )
```

## 퍼실리테이터 적용

WS2(As-Is)·WS3(To-Be)를 그릴 때, 단계를 다 모은 뒤 반드시 한 번 묻는다:
> "이 중에 **Yes/No로 갈리거나, 안 되면 앞으로 되돌아가는** 지점 있어요?"

있으면 그 단계를 **마름모**로 바꾸고, Yes·No 각각 어디로 가는지(복귀 포함) 함께 적는다. 이렇게 해야 어떤 업무가 와도 흐름이 정확해진다.
